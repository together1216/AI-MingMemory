# AI-MingMemory 数字生命项目交接文档

## 项目概述

**项目名称**：AI-MingMemory（数字生命）
**项目目标**：构建具备持久记忆、身份连续性、可迁移的AI智能体系统
**核心灵感**：钢铁侠贾维斯 (JARVIS)

---

## 一、核心概念（已确定）

### 1.1 种子层 (Seed Layer)

**定义**：让AI成为"它自己"的最小不可分割内核 = "基因层/灵魂"

**包含5个组成部分**：
1. **身份定义** - "我是谁"的基本声明、关系、边界
2. **价值观层级** - 什么是重要的、冲突时的优先级、绝对不做的事
3. **行为模式** - 遇到X情况的反应、决策风格、沟通偏好
4. **推理风格** - 如何思考、偏好什么分析方法、如何处理不确定性
5. **连续性验证** - 唯一标识符、验证"我仍是我"的机制、种子哈希

### 1.2 种子 vs 记忆

| 种子 | 记忆 |
|------|------|
| "我是谁" | "发生了什么" |
| 几乎不变 | 随时在变 |
| 删了 = 身份死亡 | 删了 = 失去履历 |
| 无法重建 | 可以重新积累 |

## 二、最新架构设计（7层）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Layer 0: 输入终端层                               │
│   Web界面 │ CLI │ API接口 │ 语音终端                                 │
│                           │                                          │
│                           ▼                                          │
│              🔗 生命绑定层 (Life Binder) ⚡ 强制关联               │
│              所有输入必须经过此层，强制绑定数字生命上下文              │
└───────────────────────────┬─────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│               Agent Loop (强制执行引擎)                               │
│   强制加载: 种子 + 记忆 + 生命Binder + 记忆管理Agent               │
└───────────────────────────┬─────────────────────────────────────────────┘
                            │
          ┌─────────────────┼─────────────────┐
          ▼                 ▼                 ▼
┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
│  Layer 3       │ │  Layer 4       │ │  Layer 5       │
│  记忆管理Agent │ │  种子层(SEED) │ │  分级记忆系统   │
│  • 主动记忆   │ │  💎 基因层/灵魂  │ │  🧠 MemGPT+   │
│  • 主动分析   │ │  • 身份定义    │ │    Nocturne    │
│  • 科学遗忘   │ │  • 价值观层级  │ │                 │
│                 │ │  • 行为模式    │ │  L1: 工作窗口  │
│                 │ │  • 推理风格    │ │  L2: 近期缓存  │
│                 │ │  • 连续性验证  │ │  L3: 深层存储  │
│                 │ │                 │ │  L4: 基因层     │
└─────────────────┘ └─────────────────┘ └─────────────────┘
```

### 分级记忆系统 (L1-L4)

| 层级 | 名称 | 存储位置 | 响应时间 | 说明 |
|------|------|----------|----------|------|
| L1 | 工作窗口 | Python deque | 毫秒级 | 最近5-10轮对话 |
| L2 | 近期缓存 | TTLCache | 百毫秒级 | 最近1-7天重要记忆 |
| L3 | 深层存储 | Nocturne URI | 秒级 | 全部历史记忆 |
| L4 | 基因层 | seed.json | 始终加载 | 核心身份、价值观 |

### 记忆分类 (URI结构)

```
identity://    → 身份定义（我是谁）
preference://  → 用户偏好（喜欢什么）
episodic://    → 对话经历（发生了什么）
semantic://    → 知识概念（学会什么）
fact://        → 事实知识（客观事实）
relation://    → 关系网络（谁和谁什么关系）
emotion://     → 情感标记（情绪状态）
```

## 三、技术选型（已确认）

| 模块 | 技术选型 | 理由 |
|------|----------|------|
| 记忆存储 | Nocturne Memory | 本地可控、URI结构、可迁移 |
| 分级检索 | MemGPT思想 | 四级渐进加载 |
| 种子层 | JSON文件 | 简单、可读、可迁移 |
| LLM | DeepSeek/MiniMax/Qwen | 用户提供API |
| Agent核心 | OpenCode二次开发 | 已有强大Agent能力 |
| 数据库 | SQLite | 作为Nocturne索引 |

### 用户核心约束（记住）

1. ❌ 不使用向量数据库方式
2. ❌ 不使用SQLite + JSON（替换现有存储）
3. ✅ 种子层 = 基因层
4. ✅ 记忆系统 = Nocturne Memory + MemGPT分级
5. ✅ 终端可以与核心耦合
6. ✅ MemGPT是补充Nocturne，不是替换
7. ✅ LLM失败时暂停并明确告知

## 六、生命绑定层 (Life Binder)

用户新增的需求 - 确保所有输入强制与数字生命关联：

```python
class LifeBinder:
    def __init__(self, digital_life):
        self.life = digital_life
        self.session_state = {}
    
    def bind(self, user_input):
        # 1. 强制加载生命上下文
        life_context = self.life.get_context()
        
        # 2. 绑定输入与生命
        bound_input = {
            "raw_input": user_input,
            "life_id": self.life.id,
            "life_context": life_context,
            "session_state": self.session_state,
            "timestamp": now()
        }
        
        # 3. 验证通过 → 放行
        return bound_input
```
